# dotbot-go v0.1.1

## 版本定位

`v0.1.1` 是在 `v0.1.0` 基础上的一次安全性与交互体验修订版本.

这一版不扩展新的配置能力, 重点是把 `link.force` 和 `clean.force` 的高风险行为做得更可控, 更可解释, 同时补齐发布文档组织.

## 主要变化

### 1. `link.force` 增加风险确认流程

- 普通覆盖行为仍然允许执行
- 命中受保护目标时, `dry-run` 会明确提示 `confirmation required`
- 交互环境下会先汇总风险项, 再进行一次 `y/N` 确认
- 非交互环境下必须显式传入 `--allow-protected-target`

受保护目标包括:

- `/`
- Home 根目录
- 当前工作目录根
- 配置文件基准目录

### 2. `clean.force` 恢复实际语义

- 默认 `clean` 只清理 dead target 位于仓库基准目录内的失效链接
- `clean.force = true` 时, 允许清理 dead target 位于仓库基准目录外的失效链接
- 如果 `clean.paths` 命中高风险根路径, 交互环境会先汇总风险项再做一次 `y/N` 确认
- 非交互环境下必须显式传入 `--allow-risky-clean`

### 3. 发布文档整理

- 发布说明统一收敛到 `docs/release/`
- `v0.1.0` 发布说明已迁移到 `docs/release/v0.1.0.md`
- 当前版本新增 `docs/release/v0.1.1.md`

## CLI 补充

这一版新增两个高风险 override 参数:

- `--allow-protected-target`
- `--allow-risky-clean`

它们只用于非交互环境下显式放行高风险操作, 不改变普通风险的默认语义.

## 升级影响

如果你从 `v0.1.0` 升级到 `v0.1.1`, 需要注意:

- 某些原本会直接执行的高风险覆盖或清理, 现在会先进入确认流程
- 在脚本或 CI 中执行高风险操作时, 需要补充显式 override 参数
